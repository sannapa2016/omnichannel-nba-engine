#  Omnichannel Next Best Action (NBA) Engine

### *AI-Driven Orchestration for HCP Engagement*

The `omnichannel-nba-engine` is a predictive analytics framework designed to optimize Health Care Professional (HCP) engagement across digital and personal channels. As seen in the repository structure, this project integrates data generation, model training, and a deployment-ready app to tell a complete story of modern "Commercial Excellence."

## The Strategic Problem: Content Saturation

Physicians are overwhelmed with generic marketing. This engine solves the "relevance gap" by predicting the **Next Best Action** (e.g., Email, Rep Visit, or Speaker Invite) based on a physician’s historical behavior and preference profile.

### Key Capabilities

* **Engagement Propensity Modeling:** Uses `train_nba_model.py` to rank potential actions based on the probability of a positive HCP response.
* **Channel Preference Detection:** Identifies "Digital First" vs. "Face-to-Face" preferences to reduce marketing fatigue.
* **Real-time Decisioning:** The `app.py` serves as a prototype interface for field reps to receive live suggestions before their next call.


## Project Architecture

Referencing the file structure in:

* **`train_nba_model.py`**: The core ML pipeline. Processes HCP interactions to determine the highest-value engagement channel.
* **`generate_hcp_data.py`**: Simulates realistic omnichannel data, including CRM logs, email opens, and sample requests.
* **`app.py`**: A Streamlit/Flask interface to visualize the NBA recommendations.
* **`STRATEGIC_VALUE.md`**: A dedicated document explaining the ROI of NBA for Pharma executives.
* **`Omnichannel_nba_dashboard.png`**: A pre-saved visualization of the model's output.

## Business Impact

* **Field Force ROI:** Sales reps focus on the right doctors with the right message, increasing "meaningful interactions."
* **Digital Efficiency:** Marketing teams stop sending low-performing emails, improving domain reputation and open rates.
* **Customer Centricity:** Aligns the Pharma brand with the doctor's preferred way of learning about new therapies.

## The Final Piece of the "Precision-Patient-360" Suite

This engine orchestrates the insights generated by your other repositories:

1. [**Oncology-RWE-LOT-Engine**](https://github.com/sannapa2016/oncology-rwe-lot-engine): Identifies where a doctor has patients failing a current therapy line.
2. [**Referral-Sense-AI**](https://github.com/sannapa2016/Referral-Sense-AI): Identifies which doctors have "hidden" rare disease patients.
3. **Omnichannel-NBA-Engine:** (This Repo) Decides **HOW** to reach those doctors—via a Rep visit or a clinical email.
4. [**Patient-Adherence-ML-Framework**](https://github.com/sannapa2016/patient-adherence-ml-framework): Triggers an NBA for a "Nurse Call" if a patient is predicted to drop off.

## Technical Setup

```bash
# Install requirements
pip install -r requirements.txt

# Generate synthetic HCP data and train the engine
python generate_hcp_data.py
python train_nba_model.py

# Launch the Rep Dashboard
python app.py

```
